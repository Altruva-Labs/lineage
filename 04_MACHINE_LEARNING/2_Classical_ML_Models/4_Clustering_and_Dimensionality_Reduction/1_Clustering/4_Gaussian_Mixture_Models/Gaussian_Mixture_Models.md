# Chapter 4.2.4.1.4: Gaussian Mixture Models (GMM)

## The Problem: Hard vs. Soft Assignments

All the clustering methods so far (K-Means, DBSCAN, Hierarchical) make **hard decisions**:

- In K-Means: every point belongs to exactly one cluster
- In DBSCAN: every point is core, border, or noise
- In Hierarchical: every point belongs to whichever cluster you cut

But real life often has **ambiguity**. Consider a customer:
- Maybe 60% of their behavior matches "budget shoppers"
- And 40% matches "premium shoppers"

They don't fit neatly into one category.

**Gaussian Mixture Models (GMMs)** say: "instead of hard assignments, give each point a probability of belonging to each cluster."

---

## What Is a Gaussian Mixture Model?

A GMM is a **probabilistic model** for clustering. It assumes the data was generated by a mixture of Gaussian (normal) distributions.

In simpler words:

> Imagine you have 3 types of customers, each with their own typical spending pattern (a bell curve). Every customer is a bit from each type. Can you figure out the 3 types and how much of each is in every customer?

Instead of saying "customer A is 100% type 1", you might say:
- Customer A is 40% type 1, 50% type 2, 10% type 3

---

## The Core Idea: Mixture of Gaussians

A **Gaussian** (normal distribution) is a bell curve defined by:
- **Mean** μ: the center (where most values cluster)
- **Variance** σ²: the spread (how wide the bell curve is)

A **mixture** is a weighted combination:

$$P(x) = w_1 \cdot N(x; \mu_1, \sigma_1^2) + w_2 \cdot N(x; \mu_2, \sigma_2^2) + ... + w_K \cdot N(x; \mu_K, \sigma_K^2)$$

Where:
- $K$ is the number of components (clusters)
- $w_i$ is the weight (proportion) of component $i$ (sums to 1)
- $N(x; \mu_i, \sigma_i^2)$ is the Gaussian with mean $\mu_i$ and variance $\sigma_i^2$

In plain English: "The data comes from a blend of K Gaussian bells. Component 1 contributes w₁ of the data, component 2 contributes w₂, etc."

---

## How GMM Learns: The EM Algorithm

GMM uses a special algorithm called **Expectation-Maximization (EM)** to learn the parameters.

### Step 1: Initialization

Guess the starting means μ₁, μ₂, ..., μₖ and weights w₁, w₂, ..., wₖ.

(Or initialize by running K-Means first, which often gives better starting points.)

### Step 2: E-Step (Expectation)

For each point, calculate the probability it came from each component.

$$P(\text{component } i | x) = \frac{w_i \cdot N(x; \mu_i, \sigma_i^2)}{\sum_j w_j \cdot N(x; \mu_j, \sigma_j^2)}$$

This is Bayes' rule: "Given that we observed x, what's the probability it came from component i?"

Example: x = 1000, and there are 3 components with means at 100, 500, 1000:

Component 1 (mean 100): very unlikely → probability = 0.05
Component 2 (mean 500): somewhat likely → probability = 0.15
Component 3 (mean 1000): very likely → probability = 0.80

So x is 80% likely from component 3, 15% from component 2, 5% from component 1.

### Step 3: M-Step (Maximization)

Update the parameters to maximize the likelihood given the current letter assignments.

New means:
$$\mu_i = \frac{\sum_n P(\text{component } i | x_n) \cdot x_n}{\sum_n P(\text{component } i | x_n)}$$

This is a **weighted average**: each point contributes to each centroid according to its probability of coming from that component.

New weights:
$$w_i = \frac{1}{N} \sum_n P(\text{component } i | x_n)$$

This is the average probability across all points.

### Step 4: Repeat

Toggle between E-step and M-step until the parameters stop changing.

## Real-World Example: Customer Segments

Suppose you have customer data (annual spending):
- Segment A (budget): mean $200, variance 50² 
- Segment B (regular): mean $800, variance 150²
- Segment C (premium): mean $3000, variance 500²
- Weights: 40% budget, 35% regular, 25% premium

A customer who spent $750 would have:
- P(budget | $750) = 0.10 (unlikely for budget shoppers)
- P(regular | $750) = 0.70 (very likely for regular shoppers)
- P(premium | $750) = 0.20 (somewhat likely for premium shoppers)

So you'd say: "This customer is 70% likely a regular shopper, 20% likely premium, 10% likely budget."

This is more nuanced than "customer is of type regular" from K-Means.

---

## Interpreting Soft Assignments

Instead of hard cluster assignments, you get probabilities:

| Customer | P(Budget) | P(Regular) | P(Premium) |
|----------|-----------|-----------|-----------|
| A        | 0.85      | 0.10      | 0.05      |
| B        | 0.02      | 0.90      | 0.08      |
| C        | 0.05      | 0.15      | 0.80      |
| D        | 0.40      | 0.40      | 0.20      |

Customer A is clearly budget-focused. Customer D is ambiguous - equally between budget and regular.

This uncertainty is **information you can use**. Maybe customer D gets marketing for both budget and regular products.

---

## Strengths

**Probabilistic Output**: Unlike K-Means ("you're cluster 1"), GMM gives probabilities ("you're 70% cluster 1, 30% cluster 2"). More informative.

**Flexible Cluster Shapes**: Each Gaussian can have its own variance. A cluster can be wide in one direction and narrow in another (elliptical).

**Well-Grounded Mathematically**: Built on probability theory. You can compute likelihoods, compare models, make principled decisions.

**Works Well When Data is Actually Gaussian**: If your data really does come from Normal distributions, GMM is optimal.

---

## Limitations

**Choose K in Advance**: Like K-Means, you must specify the number of components. Too many = overfitting. Too few = under-fitting.

**Gets Stuck on Local Optima**: EM is not guaranteed to find the global best solution. Different initializations give different results.

**Computational Cost**: More expensive than K-Means because you're tracking probabilities for every point and every component.

**Assumes Gaussian Distribution**: If your data doesn't come from Gaussians, the model is mismatched. Real data is often multimodal or has heavy tails.

**Covariance Matrix Complexity**: With many features, estimating the covariance matrix (variance) for each component requires many parameters, which needs lots of data.

---

## Soft vs. Hard Clustering Comparison

| Aspect | Hard (K-Means) | Soft (GMM) |
|--------|---|---|
| Assignment | Each point in one cluster | Probability distribution |
| Output | Discrete (1, 2, or 3) | Probabilities (0.6, 0.3, 0.1) |
| Interpretability | Simpler | More nuanced |
| Parameters | Centroids only | Means, variances, weights |
| Computational Cost | Low | Moderate |
| Mathematical Framework | Optimization | Probability theory |

---

## Connection to What Came Before

K-Means, DBSCAN, and Hierarchical clustering all treat clustering as a geometric problem: "how are points arranged in space?"

GMM treats it as a statistical problem: "what distributions generated this data?"

This shift in perspective is important:

- **Geometric**: focus on distances and neighborhoods
- **Statistical**: focus on likelihood and probability

This philosophical difference will matter more later when we work with neural networks, which also think probabilistically (softmax, cross-entropy loss).
