# Chapter 4.2.1: Introduction to Classical Machine Learning Models

Now that we understand how learning from data works, the next question is:

> what kinds of model families did classical machine learning actually use?

This section answers that.

## 1. These Are Core Families, Not the Only Families

The model groups in this section are very important,
but they are not the only ones in all of classical machine learning.

Other important families also exist, such as:

- kernel methods, like support vector machines
- ensemble methods, like random forests and boosting
- probabilistic generative models

This section will include short treatments of those families too, so you see the full range of ideas.

But for the main lineage, we focus on the most foundational and teachable families:

- linear models
- decision trees
- nearest-neighbor methods
- clustering and dimensionality reduction
- probabilistic models
- kernel methods
- ensemble methods

Why these ones?

Because together they show the main styles of classical machine learning:

- weighted linear combination
- recursive feature-based splitting
- similarity-based local prediction
- structure discovery without labels
- probabilistic reasoning
- kernel-induced nonlinearity
- model combination

## 2. Why This Section Comes Before the Learning Setups

These model families are the building blocks.

Once we understand them, we can more clearly see how they are used inside:

- supervised learning
- unsupervised learning
- reinforcement learning

So this section gives us the model toolbox first.
The next section will show how different learning setups use that toolbox.
